How Aircall Messaging Works Here

Why can't we pull historical message data?

Aircall's public REST API has no GET endpoint for messages. Unlike calls — which have GET /v1/calls — there is no equivalent for SMS history. The API only supports sending messages outbound.

Aircall API docs — Messages reference ↗

If Aircall's own dashboard shows message analytics, why can't we?

Aircall's analytics dashboard is an internal product that queries their own private database directly — it is not built on the same public REST API that third parties use. Aircall deliberately keeps this data behind their dashboard and does not expose it via API. This is a common practice: many SaaS platforms offer richer internal reporting than what they expose externally.

View Aircall's message analytics dashboard ↗

✅ Our solution: Webhook capture

Every time a message is sent or received, Aircall pushes the event to this server in real time. We store it locally and build analytics from that. The data grows automatically — no manual work needed.

SMS sent/received → Aircall pushes event → server catches it → stored in aircall-messages.json → analytics built here

🔧 One-time setup

  1. 1

    Aircall → Integrations & API → Webhook

    Search "webhook" in the integrations list

  2. 2

    Set the webhook URL

    http://203.143.26.26:3000/aircall-webhook
  3. 3

    Subscribe to these two events

    message.sent message.received
  4. ✓

Coverage limitation

Analytics only cover messages from the webhook setup date onwards. There is no way to recover messages sent before that date through any API method — Aircall does not provide an export endpoint. Use the Aircall analytics dashboard ↗ for historical data.
